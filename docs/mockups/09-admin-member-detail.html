Design Decisions - Mockup 09: Admin Member Detail

  • Two-column card grid organizes related information into scannable sections that collapse to single-column on tablet/mobile. Each card has a clear header with contextual action link (Edit, Manage, Force Sync, etc.).
  • Role hierarchy visualization uses a horizontal node-arrow pattern showing the member's current role (highlighted in blue) within the full role chain. This helps admins understand permission implications of role changes.
  • Household card highlights the primary account holder with a blue background, then lists dependents with their relationship and status. The "Add Member" link supports the household membership model.
  • HighLevel Sync card provides integration visibility at a glance: Contact ID for cross-referencing, last sync timestamp, sync health indicator (green/amber/red dot), and applied tags. "Force Sync" action for manual intervention.
  • Toggle switch for auto-renew uses native-feeling iOS-style toggle with green=on state, providing immediate visual feedback for this critical membership setting.
  • Contributions table spans full width with type badges (Donation=gold, Patron Dues=purple, Event Ticket=blue) for quick visual categorization of financial transactions.
  • Activity timeline uses a vertical line with color-coded dots (matching the dashboard activity feed colors) to show member history chronologically.
