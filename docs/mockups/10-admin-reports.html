Design Decisions - Mockup 10: Admin Reports Dashboard

  • Date preset buttons use pill-shaped toggles with the active state in blue, providing quick one-click time range filtering. YTD is selected by default as the most useful view for organizational reporting.
  • Membership table uses inline progress bars for retention % with semantic color thresholds: green above 80%, amber 60-80%, red below 60%. This enables instant visual assessment of tier health.
  • Growth column uses color-coded values: green for positive growth, red for negative. No icon clutter -- just the color and +/- sign for clean data density.
  • Campaign table highlights the top performer with a subtle gold background row and star label, drawing the eye to the most successful campaign without disrupting the table layout.
  • Campaign completion progress bars change color based on performance: green for on-track or exceeded (>75%), amber for moderate (50-75%), red for at-risk (<50%). Values exceeding 100% show the overshoot.
  • Chapter health uses a dot + text system (green=Excellent/Good, amber=Fair, red=Needs Attention) to provide a quick health assessment that combines multiple signals (member count, event frequency, campaign participation) into a single indicator.
  • Totals row in the membership table uses a heavier border and bold styling to clearly separate summary data from individual tier rows.
  • All tables support horizontal scrolling on smaller screens, with white-space: nowrap on cells to prevent data from breaking across lines and becoming unreadable.
